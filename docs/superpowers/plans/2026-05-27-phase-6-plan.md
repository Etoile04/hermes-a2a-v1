# Phase 6 Implementation Plan: Monitoring + Orchestration + Security

**Date:** 2026-05-27
**Branch:** `feat/phase-6-monitoring-orchestration-security`
**Base:** `main` (commit a44ddba)

## Overview

Phase 6 adds three major capability areas to hermes-a2a-v1:
1. **Web Monitoring Dashboard** — Real-time observability
2. **Multi-Agent Orchestration** — Agent collaboration framework
3. **Security Hardening** — Production-grade auth & audit

## Task Breakdown

### Track A: Web Monitoring Dashboard

#### A1. SSE Metrics Stream Endpoint
- **File:** `src/hermes_a2a/admin_api.py` (extend)
- Add `GET /admin/metrics/stream` SSE endpoint
- Push real-time metrics: request count, latency percentiles, active tasks, peer health
- Use existing `rate_limiter.py` metrics + task store stats
- **Tests:** `tests/test_admin_api.py` (extend)

#### A2. Static Dashboard UI
- **File:** `src/hermes_a2a/dashboard/` (new directory)
- Single-page HTML dashboard (no framework, pure HTML/CSS/JS)
- Embedded in gateway via `StaticFiles` mount at `/admin/dashboard/`
- Show: health overview, peer status cards, task timeline, metrics charts (Chart.js CDN)
- **Tests:** `tests/test_dashboard.py`

#### A3. WebSocket Live Updates
- **File:** `src/hermes_a2a/ws_broadcaster.py` (new)
- WS endpoint `ws://host:port/admin/ws` for live dashboard updates
- Broadcast: task events (created/completed/failed), peer status changes, metrics ticks
- **Tests:** `tests/test_ws_broadcaster.py`

### Track B: Multi-Agent Orchestration

#### B1. Task Decomposition Engine
- **File:** `src/hermes_a2a/orchestrator.py` (new)
- `OrchestrationEngine` class: split complex tasks into subtasks
- Subtask assignment to available peers based on skills matching
- Fan-out/fan-in pattern: distribute → collect → aggregate
- **Tests:** `tests/test_orchestrator.py`

#### B2. Skill-Based Routing
- **File:** `src/hermes_a2a/skill_router.py` (new)
- Parse peer AgentCards for skill matching
- Score peers by skill relevance + historical success rate
- Support routing rules (regex patterns, tag matching, priority)
- **Tests:** `tests/test_skill_router.py`

#### B3. Result Aggregation
- **File:** `src/hermes_a2a/result_aggregator.py` (new)
- Collect subtask results from multiple peers
- Merge strategies: concatenate, summarize, vote, custom
- Timeout + partial result handling
- **Tests:** `tests/test_result_aggregator.py`

### Track C: Security Hardening

#### C1. JWT Authentication
- **File:** `src/hermes_a2a/auth.py` (new)
- JWT token generation and validation
- Support HS256 and RS256 algorithms
- Token refresh flow
- Middleware for admin API and A2A endpoints
- **Tests:** `tests/test_auth.py`

#### C2. Audit Logging
- **File:** `src/hermes_a2a/audit_logger.py` (new)
- Structured JSON audit log for all A2A operations
- Log: timestamp, actor, action, target, result, metadata
- Rotation and retention policies
- Optional webhook delivery for SIEM integration
- **Tests:** `tests/test_audit_logger.py`

#### C3. mTLS Support
- **File:** `src/hermes_a2a/tls_config.py` (new)
- Mutual TLS configuration for peer connections
- Certificate validation and pinning
- Auto-reload on cert changes
- **Tests:** `tests/test_tls_config.py`

## Execution Order

Parallel tracks:
- **Track A** (A1→A2→A3): Monitoring dashboard
- **Track B** (B1→B2→B3): Multi-agent orchestration
- **Track C** (C1→C2→C3): Security hardening

Use `delegate_task` for parallel implementation of independent tracks.

## Acceptance Criteria

- All new tests pass (target: 320+ total)
- Dashboard renders at `/admin/dashboard/` with live data
- Orchestration demo: split a research task across 2+ peers
- JWT auth protects admin endpoints
- Audit log captures all operations
- mTLS peer connections verified
- CI passes on all 3 Python versions (3.11, 3.12, 3.13)

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| Dashboard too heavy | Keep it single-file HTML, no build step |
| Orchestration complexity | Start with simple fan-out, add patterns incrementally |
| JWT key management | Support both static secret and file-based keys |
| mTLS cert setup | Provide script to generate self-signed test certs |
